Idea General

Rating Umovies

El producto a desarrollar es una aplicación en la cual los jóvenes de 18 a 25 años puedan obtener información sobre, películas, series, libros y documentales de manera rápida y sencilla, donde se verá una pequeña reseña, puntuaciones de crítica periodística y crítica académica además de la puntuación y comentarios de la comunidad.

Objetivos Del Proyecto

Problemática

Buscar en diferentes plataformas y páginas web información sobre productos de entretenimiento, reseñas, críticas, puntuaciones y comentarios. Solución: Concentrar toda la información en una única aplicación que será alimentada por la comunidad para ampliar la base de datos y los filtros de búsqueda.

Solución

Concentrar toda la información en una única aplicación que será alimentada por la comunidad para ampliar la base de datos y los filtros de búsqueda.

Justificación

Vale la pena realizar este proyecto ya que corta los tiempos que pierden las personas al buscar recomendaciones además de que es un medio de entretenimiento de trasfondo ya que al permitir comentarios, las personas pueden debatir, compartir ideas, conocer, exponer sus proyectos, entre otras cosas a pesar de no ser un medio social.

Usuario Final

La aplicación está dirigida principalmente a jóvenes entre 18 y 25 años, ya que son altos consumidores de contenido multimedia, además de que este grupo, apoya de manera más constante a proyectos en nacimiento con contribuciones monetarias. A pesar de lo dicho, cualquier persona con un dispositivo móvil con Android superior al 4.0 y IOS superior al 7.1.2 podrán utilizarla.

Utilidad

Para obtener un retorno de inversión se pondrán pequeños anuncios dentro de la aplicación, también, un sistema, donde si el usuario desea quitarlos, podrá ejecutar un único pago para retirarlos y un sistema de donaciones, ya que no a todos les molestan los anuncios pero quieren aportar.

Requisitos

Requisitos Funcionales

  • El sistema permitirá la gestión de Clientes, bien sea directamente a la base de datos del sistema o mediante el registro de una plataforma (Facebook o Google+).
  • El sistema permitirá la gestión de contenido multimedia de su propiedad para que la comunidad pueda evaluarlo y generar críticas.
  • El sistema permitirá la gestión crítica y evaluación del contenido multimedia de los Productores.
  • El sistema permitira el ingreso seguro al aplicativo
  • El sistema permitirá la modificación de datos del Perfil del Usuario (modificar perfiles)
  • Como vendedor, quiero registrar las ventas y donaciones que lleva la plataforma para tener un informe completo de la recuperación de inversión que llevo.
  • Como vendedor, quiero poner una tarifa óptima para retirar los anuncios según los precios que manejan otras aplicaciones.
  • Como usuario, quiero ver precios de suscripción a las plataformas de streaming para evitar trasladarme de plataforma y tomar una decisión directa.
  • Como vendedor, quiero poder consultar y editar datos de los usuarios para poder solucionar algún error que se presente en ellos.
  • Como vendedor, quiero poder administrar la locación de los anuncios para evitar que tales obstaculicen alguna información.

Requisitos No Funcionales

  • Toda funcionalidad del sistema (a excepción de la carga del contenido multimedia) debe responder al usuario en menos de 2 segundos
  • El sistema debe contar con una pantalla de contacto
  • El sistema debe poseer interfaces gráficas bien formadas, modernas y agradables para todos los usuarios
  • El sistema debe tener una disponibilidad del 99,99% de las veces en que un usuario intente accederlo.
  • La interfaz de usuario será implementada para navegadores web únicamente con HTML5 y JavaScript.
  • El sistema no revelara a sus los usuario otros datos personales de los demás usuarios a diferencia del nombre de usuario.

Alcance del Sistema

  • Por el lado de costos vamos a utilizar programas gratuitos para el desarrollo del sistema, el único costo es la mano de obra que será recuperada con los anuncios que se van a integrar.
  • El proyecto se desarrollará a través de los tres cortes del semestre,donde el tiempo es suficiente para completarlo. Dicho esto, está abierto a cambios por cualquier imprevisto que aparezca en en el transcurso del desarrollo.
  • El sistema está limitado por las personas, ya que clientes de tercera edad o muy corta edad no sabrán usarla o siquiera que exista, aún así está dirigida a jóvenes-adultos.
